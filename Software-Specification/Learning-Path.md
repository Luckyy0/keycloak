# Learning Path Specification
# Đặc tả Lộ trình Học tập

## 1. Purpose / Mục đích
This document defines the learning paths tailored for different professional roles. Not all learners need to consume the 50 chapters linearly.
Tài liệu này định nghĩa các lộ trình học tập được thiết kế riêng cho các vai trò chuyên môn khác nhau. Không phải tất cả người học đều cần tiêu thụ 50 chương theo tuyến tính.

## 2. Defined Roles / Các Vai trò được Định nghĩa

### 2.1. Backend Engineer (Fresher/Mid) / Kỹ sư Backend
- **Focus / Trọng tâm**: Tích hợp Keycloak vào ứng dụng, bảo vệ API, hiểu cơ bản về Token.
- **Required Chapters / Các chương bắt buộc**: 01-10, 21-22, 44-48.
- **Outcome / Kết quả**: Có khả năng xây dựng Resource Server và Client an toàn bằng Spring Boot và React.

### 2.2. Senior Backend & Architect / Kỹ sư Backend Cấp cao & Kiến trúc sư
- **Focus / Trọng tâm**: Kiến trúc Microservices, BFF, Token Exchange, hiệu năng, phân cụm.
- **Required Chapters / Các chương bắt buộc**: Toàn bộ lộ trình của Kỹ sư Backend + Chương 11-15, 26-29, 34-37, 42-47, 50.
- **Outcome / Kết quả**: Có khả năng thiết kế hệ thống phân tán an toàn, xử lý các luồng xác thực phức tạp.

### 2.3. IAM Engineer / Kỹ sư IAM
- **Focus / Trọng tâm**: Quản trị Keycloak, Federation, Identity Broker, Custom SPI.
- **Required Chapters / Các chương bắt buộc**: 01-20, 23-25, 30-33, 42-43.
- **Outcome / Kết quả**: Trở thành chuyên gia vận hành, tùy biến và liên kết các nguồn định danh với Keycloak.

### 2.4. DevSecOps & Security Engineer / Kỹ sư Bảo mật & DevSecOps
- **Focus / Trọng tâm**: Bảo mật hệ thống, hardening, OWASP, Docker/K8s deployment.
- **Required Chapters / Các chương bắt buộc**: 01-09, 36-41.
- **Outcome / Kết quả**: Đảm bảo an toàn hạ tầng Keycloak, giám sát và phòng thủ trước các cuộc tấn công mạng.

## 3. Study Guidelines / Hướng dẫn Học tập
- Always start with the **Prerequisites** section of each module.
- Luôn bắt đầu bằng phần **Điều kiện tiên quyết** của mỗi mô-đun.
- Hands-on labs are mandatory for moving to the next level.
- Các bài thực hành trực tiếp là bắt buộc để chuyển sang cấp độ tiếp theo.

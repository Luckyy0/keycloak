# Interview Generation Rules Specification
# Đặc tả Quy tắc Sinh Câu hỏi Phỏng vấn

## 1. Scenario-based Questions / Câu hỏi Dựa trên Tình huống
- Avoid simple definitional questions ("What is X?").
- Tránh các câu hỏi định nghĩa đơn giản ("X là gì?").
- Use problem-solving scenarios: "How would you solve Y using Keycloak?", "What happens if Z fails during authentication?"
- Dùng các kịch bản giải quyết vấn đề: "Bạn sẽ giải quyết Y bằng Keycloak như thế nào?", "Điều gì xảy ra nếu Z thất bại trong quá trình xác thực?"

## 2. System Design & Architecture / Thiết kế Hệ thống & Kiến trúc
- Include system design interview questions specifically for Architect and Senior IAM roles.
- Bao gồm các câu hỏi phỏng vấn thiết kế hệ thống dành riêng cho các vai trò Kiến trúc sư và IAM Cấp cao.
- Cover topics like multi-region deployment, replication, scaling, and high availability.
- Bao gồm các chủ đề như triển khai đa khu vực, sao chép, mở rộng quy mô và tính sẵn sàng cao.

## 3. Expected Answers (Rubrics) / Câu trả lời Dự kiến (Tiêu chí đánh giá)
- Provide comprehensive rubrics showing expected answers.
- Cung cấp các tiêu chí đánh giá toàn diện cho thấy các câu trả lời dự kiến.
- Differentiate answers based on seniority (e.g., what a "Junior" vs "Senior" candidate would say).
- Phân biệt các câu trả lời dựa trên thâm niên (ví dụ: ứng viên "Junior" so với "Senior" sẽ nói gì).

## 4. Mandatory Inclusion / Bắt buộc Bao gồm
- Every single lesson file MUST include an `Interview Questions` section at the bottom.
- Mọi tệp bài học đều BẮT BUỘC phải bao gồm phần `Câu hỏi Phỏng vấn` ở cuối.
- For important or core topics (e.g., OAuth2 flows, IAM concepts, Security Hardening, Integrations), there must be AT LEAST 5 interview questions.
- Đối với các chủ đề quan trọng hoặc cốt lõi (ví dụ: các luồng OAuth2, khái niệm IAM, Tăng cường Bảo mật, Tích hợp), phải có ÍT NHẤT 5 câu hỏi phỏng vấn.

# Project Rules Specification
# Đặc tả Quy tắc Dự án

## 1. Runnable End-to-End / Có thể chạy Từ đầu đến cuối
- Every chapter must have a standalone, runnable project in its `code/` directory.
- Mỗi chương phải có một dự án độc lập, có thể chạy được trong thư mục `code/`.
- The Capstone Project (Chapter 50) must integrate Frontend (React/Angular), Backend (Spring Boot), API Gateway, and Keycloak in a single Docker Compose environment.
- Dự án chung cuộc (Chương 50) phải tích hợp Frontend (React/Angular), Backend (Spring Boot), API Gateway và Keycloak trong một môi trường Docker Compose duy nhất.

## 2. Realistic Requirements / Yêu cầu Thực tế
- Use real-world business scenarios (e.g., E-commerce, Banking, Healthcare).
- Dùng các kịch bản kinh doanh thực tế (ví dụ: Thương mại điện tử, Ngân hàng, Y tế).
- Projects must implement proper error handling and logging, not just the happy path.
- Các dự án phải triển khai xử lý lỗi và ghi nhật ký đúng cách, không chỉ là đường dẫn lý tưởng.
